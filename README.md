# Базовая часть проекта High Load. 
Предназначена для переноса (клонирования) его на сервер VPS с целью достижение 500+ RPS.  
500 requests per second — это нагрузка в 500+ запросов в секунду.

Планируется использование сервера VPS, с настройкой на нем PHP и OPcache и дальнейшей заменой PHP-FPM на Octane, после чего переход к горизонтальнму масштабированию серверов с балансировщиком нагрузки, согласно трем этапам плана ниже.

## Сервер и Инфраструктура:
Когда оптимизация кода перестает давать результат, переходите к серверу.
1. Настройте PHP и OPcache: OPcache хранит скомпилированный PHP-код в памяти, что ускоряет его выполнение в разы. Также необходимо правильно настроить пул PHP-FPM, рассчитав количество процессов под количество ядер CPU.
2. Замените стандартный PHP-FPM на Octane: Laravel Octane (на базе Swoole или RoadRunner) — это технология, которая загружает ваше Laravel-приложение в память один раз, и оно обрабатывает запросы, не перезагружаясь заново каждый раз. Это дает колоссальный прирост производительности.
3. Пересмотрите архитектуру: Если один сервер не справляется, масштабируйтесь горизонтально — поднимайте несколько серверов с приложением за балансировщиком нагрузки. БД (PostgreSQL) вынесите на отдельный мощный сервер, а для кэша и очередей используйте Redis.
